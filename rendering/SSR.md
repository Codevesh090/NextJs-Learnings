What is Server Side Rendering ?
Server Side Rendering is the process such that server builds the page first, browser later makes it interactive.In server side rendering , the Initial Real Dom is generated by the Server ,Isme Browser sirf interactivity add karta hai.


Advantages of SSR
1. Search engines now can easily index the server rendered content , because this time empty HTML is not coming , this time a fully well made HTML is coming and that is what search engine use to know what this page contains and index on basis of that.

2. User see actual HTML content right away instead of staring at a blank screen or loading spinner .



Drawbacks of SSR
1. You have to fetch everything before you can show anything which means that If a component needs to fetch data from a database or some another source , this fetching must be completed before servercan begin rendering the page .This can delay server response time to the browser ,as the server must finish collecting all necessary data before any part of the page can be sent to the client .

2. You have to load everything before you can hydrate anything.For successful hydration ,where React adds interactivity to the Server Rendered HTML ,the component tree in the browser must exactly match the server generated component tree .This means that all the Javascript for the components must be loaded on the client before you can start hydrating any of them .

3. You have to hydrate everything before you can interact with anything.React hydrates the component tree in a single pass meaning once it start hydrating ,it wont stop untill its finished with the entire tree .As a consequence all the components ust be hydrated before you can interactith any of them .

MOST IMPORTANT DIFFERENCE
══════════════════════════════════════════════════

CSR
│
├── Server sends mostly empty HTML
├── Browser downloads JS
├── React builds UI in browser
└── User waits for JS before seeing content


SSR
│
├── Server executes React first
├── Server sends READY HTML
├── Browser instantly shows content
├── JS downloads later
└── Hydration adds interactivity

Toh SSR ko samajhane ke liye hume step by step samajhana hoga yeh karta kya and kaise hai .
1. Sabse pehle ek developer code likhta hai NextJS me(as NextJs follows SSR and React follows CSR)
2. Uske baad us code ka build hota hai which means Transpilation JSX -> JS conversion
3. Aur bundle create hota hai ,But in SSR hamesha do bundle create hote hai one for client and for server.
4. Now ab developer ne yeh code likhkar ,application deploy kar di .
5. Now ab user aaya hamari website par , toh sabse pehle request jaayegi Server par to send the HTML.
6. Ab server us request ko recieve karega and it runs the page component means that Server Js bundle.
7. Kyuki bina us bundle ko run kiye toh HTML generate nahi hogi.
8. Now when we run that JS bundle ,Toh server side par re-render par useEffect nahi chalta balki ek
9. Hi baar me process hota hai . Yaani Jab JS bundle chalega toh fetch hoga pehle and data aayega .
10. Then Fill hoga states me , and then JS se HTML banega yaani "render(DOM) tree" ka creation hoga .
11. And that DOM tree at that time we call as Virtual DOM .
12. Uske baad NextJs use convert kar dega HTML string me .
13. Uske baad Server ek final HTML document banayega jisme HTML code ke saath script tag bhi daalega.
14. Taaki jab yeh HTMl file Browser par jaaye and execute ho , toh yeh script tag chale.
15. And Client-side Js bundle run kare . Now , when prepared Final HTML , server sends the browser(client)
16. Ab browser recieves the HTML file from the server and Browser ab us HTML ko parse karke Dom created.
17. Now, browser immediately sees actual content and paints UI FAST .Now, user can see the content.
18. But abhi tak us page me interactivity add nahi hui hai ,kyuki interactivity ka feature us client Ke
19. Js bundle me hai ,jab wo run hoga tabhi us page par interactivity aayegi .
20. Browser dekhega us bundle.js ko and the server se us client ke bundle.js ko mangega 
21. Server us file ko browser ko dega and then browser us file ko execute karna shuru kar dega .
22. Toh sabse pehle kya hoga ki React initialize hoga .
23. Now,ab kya hoga ki ek virtual DOM banega by executing that bundle.js file of client side
24. And comparison hoga If { Real Dom(Server given) = Virtual Dom(Client given) } then kya hoga ki
25. React us Real Dom me interactivity attach kar dega like button clicks and etc.. thing and now
26. And now the whole website comes into life with HTML(UI) + Js(interactivity) .That's it.
27. But listen if { Real Dom(Server given) != Virtual Dom(Client given) } then it cause hydration error.
28. And also DOM me Js interactivy nahi aati hai , only HTML skeleton aata hai.
29. Now ab website is fully alive, Now let say some user clicked some button which cause any state change
30. Toh ab again same like CSR re-render hoga and virtual DOM banega through that client side JS and then
31. Same like CSR Real Dom se comparison and then diffing and then whatever the changes it applies .
32. Yeh Hydration wala process sirf for the first time hota hai and then same diffing process hota hai .
33. That's how , Server Side Rendering works .
