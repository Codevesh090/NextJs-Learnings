Now,we will understand about the concept of "Metadata" in NextJs

<What is Metadata ? What problem does it solves ? How it is used ?>
Metadata API in NextJs is a powerful feature that lets us define Metadata for each page.
Metadata ensures our content looks great when shared or indexed by search engines.
Yaani metadata se google yaa browser ko pata chalata hai yeh webpage kis cheez ke baare me hai and also metadata is used when we are sharing our webpage with anyone.


<Two ways to handle Metadata in layout.tsx and page.tsx files?>
1. export a static metadata object 
2. export a dynamic generateMetadata function

Yaani metadata ko hum do taareke se define kar sakte hai kisi bhi webpage ke liye .



Some Metadata rules:
1. Both layout.tsx and page.tsx an export metadata.
2. Layout metadata applies to all its pages ,while page metadata specific to that page.
3. Metadata follows a top-down order ,starting from the root level
4. When metadata exists in multiple places ong a route,they merge together with the page metadata overiding layout metadata or matching properties .
5. Every route has it single metadata.

Now, we will understand "Working with dynamic metadata export"


Dynamic metadata, static metadata jaisa hi hota hai in terms of structure and usage.
Lekin dono ke beech ek key difference hai:

* Static metadata ek baar define hota hai aur har page ke liye same rehta hai.
* Dynamic metadata ka purpose ye hai ki metadata har page ke context ke hisaab se change ho sake.

Instead of having one fixed metadata for all pages, hum ek aisa system use karte hain jahan metadata same structure follow karta hai, but usme variables hote hain.

Ye variables automatically current page ko identify karte hain (jaise route params, data, etc.), aur uske basis par metadata ko customize kar dete hain.

👉 Matlab:

* Metadata ka format same rehta hai
* But content dynamic hota hai
* Aur har page apna context-aware metadata generate karta hai

We will understand through an example that we used here:

In page.tsx of [productId]

// 🔥 Dynamic Metadata Function
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { profileId } = params;

  return {
    title: `Profile ${profileId}`,
    description: `This is the profile page of user ${profileId}`,
  };
}

ONE MORE IMPORTANT THING :

----------------------------------------------------------------------------------------
For nested objects, <Next.js does not do a deep merge — the whole object gets replaced:>

// app/layout.tsx
export const metadata = {
  openGraph: {
    title: "My App",
    description: "OG desc",  // ← you'd expect this to survive
    siteName: "My Site"       // ← you'd expect this to survive
  }
}


// app/dashboard/page.tsx
export const metadata = {
  openGraph: {
    title: "Dashboard"  // ← replaces the ENTIRE openGraph object!
  }
}

---------------------------------------------------------------------------
Result — description and siteName are lost:
{
  openGraph: {
    title: "Dashboard"  // <the whole openGraph from layout is gone!>
  }
}

---------------------------------------------------------------------------
To <avoid> this, you need to manually spread the parent:

import type { Metadata } from 'next'

const baseOpenGraph = {
  siteName: "My Site",
  description: "OG desc"
}

// app/dashboard/page.tsx
export const metadata: Metadata = {
  openGraph: {
    ...baseOpenGraph,        // manually bring in the base
    title: "Dashboard"       // then override what you need
  }
}
